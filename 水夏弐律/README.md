## 游戏「水夏弐律」

该文档是关于 Galgame「水夏弐律」的序列码校验的破解记录，只是出于个人兴趣和逆向学习，不直接提供游戏资源。

## 破解思路

首先，游戏比较老，大约是 2011 年左右的。因为比较老，所以没什么复杂的壳、加密算法等等，作为练手是非常合适的。

我也不知道该从何说起，总之，第一步，打开游戏。

让游戏程序正常运行，游戏界面出来之后，这时候点击游戏界面，就会弹出窗口，要求我们输入序列号。点击输入框，会发现输入法会变为英式/美式键盘，也就是说只能输入数字、字母、简单的合法字符。另外，可能还会注意到，无论怎么输入，OK键都是未激活状态。这是当然了，因为程序校验逻辑就是必须要输入完全正确的序列号，OK键才会被激活。

一开始，我做了一些初步输入测试，没什么用。于是，直接打开 x32dbg 开始分析。

把游戏拖进去，然后直接F9，让它运行到游戏程序出来，然后点击游戏界面，弹出窗口。

这时候就去句柄选项卡的位置刷新一下，通过 OK 键按钮的句柄跳转到对应代码块。

可以尝试修改逻辑，让 OK 键激活，虽然没什么用。

然后 Ctrl + F9，让当前代码块执行到 ret。

继续按 F8，来到校验函数前，接下来这里就是关键的部分了。

下面还原游戏校验逻辑，大体上分为 4 个部分：

1. 输入的序列号长度必须为 11 位，且只能是数字+字母组合。
2. 第二步会将序列号根据映射表进行变换。
3. 第三步是拆分变换后的序列号(拆分为 2-5-4 三段，2+5+4=11)，以及取前 7 位计算 CRC + 取模运算，具体如下：
    1) 第一段是固定的: "DW"（这个要分析到最后才会发现）；
    2) 第二段调用函数计算的结果值必须要小于 10000000 且能被6整除；
    3) 第三段是 CRC 校验结果值，它等于第一段+第二段的字符串进行 CRC 计算结果；
4. 最后会检查依次第三段、第二段、第一段的值是否合法。

整个逆向过程就这些，整体来说是挺简单的，不过如果对汇编以及工具掌握还不够熟练，还是会费一番功夫的。

最后，简单说明一下，要熟练掌握工具，这样才能快速定位到关键的代码块。比如，要找到 OK 键的句柄来跳转到关键汇编代码块、测试大概是哪个 API 在不停的获取输入框中的字符串。
